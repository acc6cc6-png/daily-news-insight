Deployment Notes

Daily News Insight

这是一套“抓取原始新闻 -> 生成静态 JSON -> GitHub Pages 发布”的静态部署方案。

主简报 查看全量动态、重点排序与策略判断 源数据监控 查看各板块抓到的原始标题与来源 部署说明 更新频率、数据结构与发布路径

当前发布方式

GitHub Actions 定时跑脚本，把生成结果写到 `docs/data`，然后 GitHub Pages 直接发布。

当前默认会在北京时间 07:05、12:30、17:05、21:35 自动刷新，也支持手动触发。

Architecture

站点结构

站点前端是纯静态页面，首页只读取 `docs/data/latest/digest.json` 和历史快照，不依赖后端服务。

抓取层会聚合中文快讯源和国际 RSS 源，然后按板块做去重、全量汇总、重点排序、周期判断和联想链路整理。

Update Flow

更新流程

  1. GitHub Actions 定时运行 `scripts/daily_digest_v2.py`。
  2. 脚本抓取原始新闻，生成 `docs/data/latest/digest.json` 与历史归档。
  3. 同一个 workflow 会把 `docs/` 打包并部署到 GitHub Pages。

Secrets

密钥与本地运行

如果仓库设置了 `OPENAI_API_KEY`，脚本会优先使用更正式的编辑层输出；如果没有，也能退回到规则模板模式继续生成页面。

本地运行时可以在仓库根目录的 `.env.local` 中放入密钥；公开仓库里不要提交这个文件。
