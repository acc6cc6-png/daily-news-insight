Setup Guide

说明 / 部署

Architecture

整体模式

这个站点采用“脚本定时生成 JSON，GitHub Pages 直接展示静态页面”的模式。

因此你不需要单独买服务器，也不需要维护数据库。内容更新全部由 GitHub Actions 完成。

OpenAI Key

OpenAI Key 怎么接

  1. 在 OpenAI 后台创建项目级 API Key，不要把 Key 写进仓库文件。
  2. 本地调试时用环境变量 `OPENAI_API_KEY`。
  3. GitHub 上运行时，把同一个 Key 配到仓库的 Actions Secret 里。
  4. 可选再加一个 `OPENAI_MODEL`，默认当前项目会用 `gpt-5-mini`。

GitHub

GitHub Actions 和 Pages

  1. 仓库里已有工作流文件 `.github/workflows/daily-digest.yml`。
  2. Pages 选择 `main` 分支的 `docs` 目录。
  3. Actions Secret 里至少配置 `OPENAI_API_KEY`。
  4. 手动触发一次工作流后，站点就会有第一版正式数据。
