Positioning

站点定位

站点的目标不是做“新闻列表搬运”，而是把从昨收后到现在的跨市场信息压缩成更像投研晨会的结构：全量动态、重点主线、周期判断、策略建议和联想链路。

其中“重点新闻”不再简单按时间排序，而是先做筛选、聚类和主线重写，尽量贴近市场实际定价顺序。

Workflow

更新流程

  1. GitHub Actions 定时运行 `scripts/daily_digest_v2.py`。
  2. 脚本抓取新闻源，生成 `docs/data/latest/digest.json` 与原始快照。
  3. 同一个仓库直接通过 GitHub Pages 发布 `docs/` 目录。
  4. 前端页面只读 JSON，因此上线稳定、成本低，也方便回溯历史快照。

Deployment

部署方式

你现在这套就是 GitHub 免费托管思路：代码放仓库，任务放 Actions，页面放 Pages。前端不暴露 API key，Key 只放在仓库 Secrets 或本地 `.env.local`。

如果后续要继续扩展，也可以加更多板块、更多新闻源，或者把重点排序算法继续往“A股开盘前最该看什么”这一套去收紧。
